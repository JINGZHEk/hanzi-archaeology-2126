# 汉字考古 2126

> 中华文字探源计划 —— 一场关于网络梗词语义修复的互动实验。

当"栓 Q"不再是感谢，当"问薪吾溃"取代了"问心无愧"，当"命运的齿轮"成为万能开头……我们决定做一件事：**把被烂梗消解的厚重底蕴，重新挖出来。**

## 这是什么

汉字考古 2126 是一款纯前端的网页小游戏。玩家扮演"梗词考古学家"，在漫天飞舞的文字粒子中，找出属于目标梗词的关键字符，完成拼铸，最终揭示每个梗词被扭曲前后的真实含义。

游戏不是单纯的知识问答，而是一次关于语言退化的反思——你越了解一个词原本的意思，就越能感受到它被滥用后的荒诞。

## 玩法

游戏分为三个阶段：

**阶段一 · 梗词淘金热** —— 限时内从漂浮的文字粒子中点击收集目标汉字，填入残缺的梗词槽位。点错会冻结屏幕，考验眼力和反应。

**阶段二 · 词元拼铸** —— 将收集到的字元拖拽到正确位置，完成梗词的最终拼合。

**阶段三 · 语义觉醒** —— 展示梗词的本意溯源与扭曲含义对比，以及它被滥用的典型场景。

## 特性

- **20+ 梗词关卡**，覆盖互联网黑话、打工人文化、二次元空耳、方言谐音等多个主题
- **四级难度**（简单 / 普通 / 困难 / 地狱），逐级解锁
- **隐藏关卡**，集齐主题组后解锁特殊遗迹
- **字库图鉴**，通关后解锁每个梗词的语义考古详情
- **计分与连击系统**，追求零失误与高分
- **角色干扰机制**，闯关过程中会有角色随机捣乱
- **纯前端实现**，无需后端，打开 `index.html` 即可游玩

## 技术栈

- 原生 HTML + CSS + JavaScript（ES Modules）
- 无任何框架依赖，零构建工具
- 基于 `requestAnimationFrame` 的粒子动画系统
- 拖拽交互使用 Pointer Events API
- 本地存储使用 `localStorage` 保存进度

## 项目结构

```
hanzi-archaeology-2126/
├── index.html              # 入口页面
├── src/
│   ├── main.js             # 游戏主逻辑（状态机、关卡流程、计分）
│   ├── levels.js           # 关卡数据（梗词、难度、主题、语义文本）
│   ├── character.js        # 干扰角色定义
│   ├── characterManager.js # 角色管理器
│   ├── storage.js          # 本地存储（进度、成就）
│   └── theme.js            # 主题与视觉特效
├── styles/
│   └── app.css             # 全局样式
├── assets/                 # 静态资源
├── background/             # 背景图片（11 张）
└── character/              # 角色立绘
```

## 本地运行

直接在浏览器中打开 `index.html` 即可。如果需要本地服务器：

```bash
# 使用 Python
python -m http.server 8080

# 使用 Node.js
npx serve .
```
